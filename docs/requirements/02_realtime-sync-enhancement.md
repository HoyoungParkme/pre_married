# REQ-02: 실시간 동기화 + 고도화

- 요청일: 2026-04-15
- 상태: 구현 중
- 스프린트: 7~10

## 요청 원문
> 이 프로젝트는 깃허브에서 무료로 돌아갈 수 있는 사이트야. 두명에서 쓰는거고 그냥 사용자가 입력하면 변수가 자동으로 바뀌게 하고 싶어. 나중에 다시들어와도 마지막으로 변경한 그대로.

## 핵심 목표
- 두 사용자가 각자 기기에서 접속하여 같은 데이터를 실시간 공유

## 사용자 시나리오
1. 사용자 A가 폰에서 예산을 수정하면, 사용자 B의 PC에서 실시간 반영
2. 브라우저를 닫았다 다시 열어도 마지막 변경 상태 유지
3. 오프라인에서도 로컬 데이터로 동작, 재연결 시 자동 동기화

## 기능 목록
| # | 기능 | 우선순위 | 설명 | 상태 |
|:--|:--|:--|:--|:--|
| F-1 | Firebase 실시간 동기화 | 필수 | 방 코드로 두 사용자 연결, 실시간 데이터 공유 | 구현 중 |
| F-2 | D-day 카운트다운 | 필수 | 결혼식 날짜 설정 + D-day 표시 | 미시작 |
| F-3 | 지출 내역 기록 | 필수 | 실제 지출 기록 + 예산 대비 비교 | 미시작 |
| F-4 | UI/UX 개선 | 보통 | 접근성, 토스트 UI, 번들 최적화 | 미시작 |

## 기술 결정
- 데이터 저장소: Firebase Realtime Database (무료 Spark 플랜)
- 동기화: Zustand subscribe + Firebase onValue (기존 스토어 수정 없음)
- 인증: 없음 (방 코드가 사실상 비밀번호 역할)
